História dos computadores

(Voltar topo)

Falando um pouco da história por trás de como houve a evolução desde os primeiros computadores até os dias atuais. Para isso voltamos na década de 40, mais especificamente em 1944, surgiu o primeiro computador o Harvard Mark I. Apesar que naquela época este não se parecia nem um pouco com os dias atuais, pois além de uma estrutura totalmente complexa em relação ao seu tamanho e assim podendo preencher uma sala inteira com suas dimensões e seus processos mecânicos.

Em 1946 com o Colossus na Alemanha criado na segunda guerra mundial com objetivo de realizar comunicação por criptografias e outros cálculos relacionados a guerra. Também neste mesmo ano o ENIAC, esses computadores de primeira geração eram maquinas extremamente robustas e pesadas na época.

Na segunda geração entre 1959 a 1965 houve um passo na linha evolutiva dos computadores passando assim de maquinas com válvulas para transistores. Com esse avanço apesar de ainda grandes essas maquinas, aqui elas já se tornam um pouco mais portáteis podendo assim serem movimentadas de uma sala para a outra, onde antes alguns computadores passam a pesar até 30 toneladas, nessa geração passaram a pesar 1 tonelada ou menos conforme fonte no Wikipédia

Com o avanço tecnológico por volta de 1965 a 1970 deu-se o início a terceira geração de computadores neste com a inovação e implementação dos circuitos integrados, como transistores, capacitores e resistores em outras palavras o chips. Tornando assim mais portáteis e comuns.

Por fim na quarta geração os circuitos integrados passaram a serem micro-processadores de 1971 até os dias atuais, com isso não só temos os computadores portáteis como também temos eles de bolso, estando presentes no nosso dia-a-dia.

O que são Clients?

(Voltar topo)

Cliente quem sempre inicia a comunicação com o servidor. O cliente mais conhecido é o navegador. Quando acessamos um site pelo navegador, uma requisição é disparada pela internet para o computador no qual esse site está hospedado. O cliente-side ou client-side scripting, é uma linguagem que é executada no lado cliente, ou seja, no computador do próprio usuário, e por isso é usada nas situações em que a linguagem server-side não tem alcance.

O que são Servers?

(Voltar topo)

A principal função de um servidor web é permitir que usuários acessem aplicações, serviços e dados hospedados através da internet. Ele atua como um intermediário entre um equipamento cliente, que geralmente usa um software como um navegador web, e o servidor que armazena o conteúdo desejado. Quando um usuário faz uma solicitação de conteúdo, o servidor recebe essa solicitação e a encaminha para o servidor de origem.O servidor de origem processa a solicitação e retorna o conteúdo solicitado para o servidor web, que por sua vez envia a resposta de volta para o cliente.

Server-side, ou literalmente "lado do servidor", também conhecido por back-end, é um termo usado para designar operações que, em um contexto cliente-servidor, são feitas no servidor, não no cliente.

Como é uma estrutura básica de um HTML?

(Voltar topo)

A primeira tag !DOCTYPE html presente em nosso código é necessária para indicar aos navegadores que o documento presente se trata de um arquivo HTML. Essa lógica precisa estar sempre presente no começo dos arquivos. O representa o início dos elementos de uma página. Todos os demais elementos precisam ser descendentes dessa tag, seja diretamente ou indiretamente.

A tag de cabeçalho Head contém o conteúdo que é visualizado pela máquina, tal como os scripts, links externos e metadados. Não é um elemento visual.

O Body uma das tags mais populares e importantes do HTML, dentro dela adicionamos os conteúdos que podem ser visualizados pelo usuário final. É uma tag única e pode ser considerada a tag pai de todos os elementos de conteúdo.

Sendo assim a estrutura básica do HTML é composta das tags !DOCTYPE, head e body. Obviamente, precisamos de mais tags para que a nossa página possua algum conteúdo, porém internamente o HTML é composto e executado por essas três principais tags. De acordo com a W3C e também podemos aprender muito também com a w3schools

Entendendo a diferença entre listas ordenadas e não ordenadas

(Voltar topo)

As listas ordenadas ou numeradas são usadas para indicar alguma sequência ou numeração.

As tags usadas para criar uma lista não ordenada são ol e li. A tag ol é usada para definir a lista e a tag li é usada para cada item da lista.

As listas não numeradas são usadas para listar itens, sem se preocupar com sua sequência. Chamamos de lista de marcadores apenas.

.As tags usadas para criar uma lista não ordenada são ul e li. A tag ul é usada para definir a lista e a tag li é usada para cada item da lista.
